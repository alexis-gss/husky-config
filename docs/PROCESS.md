# Process

This package provides a generic configuration to validate commit messages using Husky, which automatically initializes during installation.

## Installation

After installing this package, the `postinstall` command will be executed:
```bash
node setup.js && [ -f $INIT_CWD/package.json ] && cd $INIT_CWD && npx husky install || true
```
This command deploys all files required for the package to work properly through the `setup.js` script and initializes Husky in the current project.

Here is the list of files generated by the `postinstall` command:

| File          | Path                                                                                                                                 |
|---------------|--------------------------------------------------------------------------------------------------------------------------------------|
| Configuration | `commitlint.config.cjs` (created by setup.js, inherits the package configuration defined [here](#configuration-commitlintconfigcjs)) |
| Husky         | `.husky/_/` (created by Husky initialization)                                                                                        |
| Hook          | `.husky/commit-msg` (created by setup.js, more details [here](#hook-commit-msg))                                                     |

## Configuration commitlint.config.cjs

This configuration file defines a total of 4 rules to apply and validate:
- `subject-full-stop`: Never end the commit subject with a period:
  ```javascript
  "subject-full-stop": [2, "never", "."],
  ```
- `type-valid` (custom rule): validates that the commit type is within an allowed list:
  ```javascript
  "type-valid": [2, "always"],
  ...
  "type-valid": (parsed) => {
     const { header } = parsed;
     const typeRegex = /^(build|chore|ci|docs|feat|fix|perf|refactor|revert|style|test):/;
     if (!typeRegex.test(header)) {
       return [false];
     }
     return [true];
   },
  ```
  List of allowed types here: [commitlint](https://github.com/conventional-changelog/commitlint/tree/master/@commitlint/config-conventional#type-enum).
- `emoji-valid` (custom rule): validates the presence of an emoji or a gitmoji code:
  ```javascript
  "emoji-valid": [2, "always"],
  ...
  "emoji-valid": (parsed) => {
     const { header } = parsed;
     const emojiRegex = /:\s+([\p{Emoji}]|:\w+:)\s+/u;
     if (!emojiRegex.test(header)) {
       return [false];
     }
     return [true];
   },
  ```
  List of allowed gitmoji codes here: [gitmoji.dev](https://gitmoji.dev/).
- `message-valid` (custom rule): validates that the commit message content is between 3 and 255 characters:
  ```javascript
  "message-valid": [2, "always"],
  ...
  "message-valid": (parsed) => {
     const { header } = parsed;
     const emojiRegex = /:\s+([\p{Emoji}]|:\w+:)\s+(.+)$/u;
     const match = header.match(emojiRegex);
     const message = match ? match.trim() : "";
     if (!message || message.length < 3 || message.length > 255) {
       return [false];
     }
     return [true];
   },
  ```

## Hook commit-msg

Hook executed before each commit to validate the commit message:

```sh
#!/bin/sh

npx --no -- commitlint --config node_modules/alexis-gss-husky-config/commitlint.config.cjs --edit $1
```

- `#!/bin/sh`: indicates shell script interpretation,
- `npx --no -- commitlint`: runs the commitlint validator,
- `--config node_modules/alexis-gss-husky-config/commitlint.config.cjs`: uses the package configuration,
- `--edit $1`: validates the commit message (`$1` = message).

## Behavior

- Examples of **valid** commits:
```bash
git commit -m "chore: ⬆️ upgrade composer dependencies"
git commit -m "fix: 🚨 resolve php errors"
git commit -m "docs: 📝 set documentation of registries"
git commit -m "refactor: ♻️ clean HandleInertiaRequests adin to keep only the essentials and move what is necessary into the package"
```
- Examples of **invalid** commits:
```bash
git commit -m "fix some errors"
# ❌ Missing emoji

git commit -m "fix: resolve some errors."
# ❌ Message ends with a period

git commit -m "bugfix: 🐛 resolve some errors"
# ❌ Invalid type (not in allowed list)

git commit -m "fix: 🐛 ok"
# ❌ Message too short
```
- To test the validation, use the following command:
```bash
echo "test message" | npx commitlint
```
